MCP AUTO-MEMORY SETUP

CONFIGURATION:

Add to your MCP client config:

{
  "mcpServers": {
    "shared-memory": {
      "command": "npx",
      "args": ["shared-memory-mcp", "${workspaceFolder}/.ai-context"],
      "env": {
        "ANTHROPIC_API_KEY": "your-key"
      }
    }
  }
}

AUTOMATIC BEHAVIOR:

The AI will now automatically:

1. LOAD MEMORY AT START
   - Reads memory://instructions resource
   - Gets persistence behavior rules
   - Loads INDEX with memory_search()

2. SAVE DURING WORK
   - After making decisions → memory_append()
   - After implementing features → memory_append()
   - After fixing bugs → memory_append()
   - After important discussions → memory_append()

3. CHECKPOINT PERIODICALLY
   - Every ~50 messages
   - Summarizes progress
   - Saves key information

4. CROSS-SESSION CONTINUITY
   - New conversation starts
   - AI loads memory automatically
   - Continues from where you left off

HOW IT WORKS:

Resources (auto-exposed to AI):
- memory://instructions - Behavior rules
- memory://INDEX - Core memory
- memory://auth - Auth topic
- memory://api - API topic
- ... all topics available

Prompts (AI can invoke):
- session-start - Initialize with memory
- save-decision - Save specific info
- periodic-save - Checkpoint progress

Tools (AI calls as needed):
- memory_search() - Find relevant context
- memory_append() - Save new info
- memory_update() - Modify existing
- memory_delete() - Remove outdated

COMPACT FORMAT:

AI saves in token-efficient MDL:

Instead of:
"We implemented JWT authentication with 15 minute access tokens and 7 day refresh tokens using the jsonwebtoken library"

Saves:
"auth: JWT tokens(15m/7d) | lib:jsonwebtoken"

This prevents context bloat while maintaining full information.

TESTING:

Session 1:
You: "Use PostgreSQL for database"
AI: [saves automatically]
  memory_append({file: "INDEX.mdl", section: "STACK", content: "db: PostgreSQL"})

You: "JWT tokens expire in 15 minutes"
AI: [saves automatically]
  memory_append({file: "auth", section: "IMPLEMENTATION", content: "JWT: 15m access"})

Session 2 (NEW conversation):
You: "Continue building the app"
AI: [loads memory automatically]
  memory_search({query: "current stack and decisions"})
AI: "I see we're using PostgreSQL and JWT auth with 15m tokens..."

RESULT:

Infinite conversation continuity without context window saturation.
